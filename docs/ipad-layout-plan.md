# iPad レイアウト対応 計画書

## 現状と原因
- iPad シミュレータで表示すると、黒背景に iPhone 縦長レイアウトが中央に表示される。
- 画面幅を固定した縦レイアウトのままレンダリングされており、iPad の広い領域を使っていない。
- Expo Go ではなく dev client でネイティブモジュールは動作しているため、表示崩れはレイアウト起因。

## 目標
- iPad（縦・横）での見やすいレイアウトを提供する。
- 共通スタイルは維持しつつ、iPhone と iPad でレイアウトがレスポンシブに変化する。

## 実装計画（チェックリスト）
### 0. 事前準備
- [ ] 変更対象画面の特定：ログイン/登録、ダッシュボード、主要フォーム（ログ入力、設定系）。
- [ ] シミュレータの検証デバイスを決める：iPad Pro 13-inch (M4) 縦・横。

### 1. レイアウトの土台をレスポンシブ化
- [ ] コンテナの maxWidth をブレイクポイントで可変にする（例：`min(540px, 90vw)` → iPad では 720–840px 程度に拡張）。
- [ ] 横方向のパディングを `vw` ベースにし、iPad で左右に十分な余白を確保。
- [ ] 縦横比が崩れないよう、背景グラデーションやカードの角丸を維持。

### 2. iPad 用のスタイル分岐
- [ ] 画面幅/デバイスタイプを判定（`useWindowDimensions` + `Platform.OS === 'ios'`）して iPad ブレイクポイントを導入。
- [ ] iPad ではフォーム幅を広げ、文字サイズ/行間を若干拡大（読みやすさ向上）。
- [ ] ボタン高さ・タップ領域を16–20pt以上に統一。
- [ ] 背景を全画面に敷く（黒余白をなくす）：SafeAreaViewで全面を覆い、カードを中央配置。

### 3. 主要画面の個別対応
- [ ] ログイン/登録：カード幅・入力フィールド幅を拡張、説明文の行折り返し調整。
- [ ] ダッシュボード：カードを2列表示に切替えるか、中央寄せで広めの1列にするかを選択（iPadブレイクポイントで分岐）。
- [ ] ログ一覧/詳細：モーダル幅・画像プレビューの最大幅を端末幅に応じて拡大。
- [ ] 設定画面：セクションのパディング・行間をiPad向けに広げる。

### 4. 共通コンポーネントの調整
- [ ] グローバルな `Layout` / `Container` コンポーネントにブレイクポイント付きの幅・パディングを追加。
- [ ] テキストスタイルの `lineHeight` と `fontSize` を iPad 時に +1〜2pt 調整。
- [ ] スクロールビューの余白（`contentContainerStyle`）を画面幅に応じて設定。

### 5. 動作確認
- [ ] iPad Pro 13-inch 縦/横で主要フローを確認（ログイン→ホーム→ログ入力→設定）。
- [ ] iPhone でも崩れていないことを確認（同一コードでレスポンシブになっていること）。
- [ ] 必要ならスクショを保存（Simulator の File > Save Screen Shot）。

### 6. リリース手順
- [ ] 変更後、開発ビルド（dev client）で iPad 確認。
- [ ] 本番用 EAS ビルドを実行し、TestFlight で最終確認。
- [ ] スクリーンショット差し替えが必要なら App Store Connect にアップロード。

## 備考
- レイアウト崩れは機能ではなくUIの広さが原因のため、レスポンシブ対応で解消する。
- Expo Router/React Native のスタイルのみで対応可能（ネイティブ改修不要）。
